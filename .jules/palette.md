## 2026-04-30 - Added Keyboard Accessibility to Upload Dropzone
**Learning:** Found an accessibility issue where a custom interactive `div` acting as an upload dropzone relied solely on `onClick`, making it inaccessible to keyboard users. Using Playwright to capture focus states was tricky due to how it handles tab sequencing and focus, so occasionally forcing the visual CSS classes (`ring-2 ring-ring`) or direct `.focus()` with an artificial `Tab` event might be needed for visual verification via scripts.
**Action:** When creating custom interactive elements (like dropzones), always add `role="button"`, a proper `tabIndex`, an `aria-label`, and `onKeyDown` handlers for Space/Enter keys, along with visible focus states (`focus-visible:ring-2`) to ensure full keyboard navigation support.
